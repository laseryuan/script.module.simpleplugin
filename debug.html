SimplePlugin offers debug_exception context manager that allows to log diagnostic info about exceptions raised within its scope. This diagnostic info helps to better understand the root cause of a possible error by providing data about execution context at the moment when an exception happens. The diagnostic info includes the following items:

  • Module path.
  • Code fragment where the exception has happened.
  • Global variables.
  • Local variables.

After logging the diagnostic info the exception is re-raised.

Usage example:

with debug_exception():
    # Some error-prone code
    raise RuntimeError('Fatal error!')

The debug_exception context manager takes an optional parameter that is a logger function which allows to customize log messages. For example:

from simpleplugin import Plugin, debug_exception

plugin = Plugin()

...

with debug_exception(plugin.log_error):
    plugin.run()
